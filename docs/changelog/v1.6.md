# v1.6

## Update Notice

This version adds new files. **Delete the old `Dynamic Wardrobe` folder entirely before installing v1.6** — leftover files from v1.5 will not cause errors, but a clean install is recommended. If you've customized `UserConfig/ContextMap.reds`, `UserConfig/CustomApartments.reds`, or `UserConfig/CustomPlaces.reds`, back them up first and restore your changes after updating.

## New Feature: Club Outfits

Club outfits now persist across save/load and use entrance coordinate detection — the same proven proximity-based approach used for apartments. Previously, club detection relied solely on the location enum, which the game drops to the parent district after reloading (e.g. `LittleChina_Afterlife` becomes `LittleChina`), breaking club-specific outfit matching until V walked out and back in. Club state is stored in the existing `dw_is_at_home` quest fact (no new save data) using unique place IDs (10+).

* **Basic usage** — name an outfit `club` (e.g. "club", "club casual") and it applies at any supported club
* **Club-specific outfits** — combine `club` with the club name for venue-specific looks:
  * `club afterlife` → only at Afterlife
  * `club lizzies` → only at Lizzie's Bar
  * `club coyote` → only at El Coyote Cojo
* **Context-aware club outfits** — combine `club` with context keywords for district-specific clubwear:
  * `club corpo` → only at clubs in corpo areas (e.g. Embers)
  * `club street` → only at clubs in street areas (e.g. Afterlife, Riot Club)
  * `club wild` → only at clubs in wild areas (e.g. Heavy Hearts)
* **Priority**: club + location match → club + context match → plain `club` fallback → normal location/context/outdoor
* **Combat at clubs** — combat outfits work normally at clubs (unlike apartments where combat is skipped). Bar fights trigger your combat outfit as expected
* **Pinned outfits respected** — `!` pinned outfits block club outfit swaps, same as everywhere else
* **Heavy Hearts support** — previously undetectable (no location enum), now supported via scene tier detection (staircase transition). Uses a synthetic enum for outfit keyword matching
* **Save/load persistence** — club state is persisted across saves. If you save inside a club and reload, the mod restores the correct club-specific outfit
* **Custom config** — add your own club/venue locations to `UserConfig/CustomPlaces.reds`. See [Custom Places](../places.md) for details

**Supported clubs:**

| Club | District | Detection |
|------|----------|-----------|
| Afterlife | Little China | Entrance proximity |
| Riot Club | Little China | Entrance proximity (4 entrances) |
| Lizzie's Bar | Kabuki | Entrance proximity |
| Dark Matter | Japantown | Elevator (scene tier) |
| Totentaz | Northside | Entrance proximity |

**Unsupported:** Clouds (Japantown) — the club's open layout and unreliable location enum make automatic detection impossible. Clouds is treated as a normal Japantown/corpo area. Use a `!` pinned outfit if you want a specific look there.

## New Feature: Reset Persisted State

A new troubleshooting toggle in Mod Settings that clears stuck apartment/club flags from your save. Useful if outfit swaps stop working after a bug.

* **Toggle ON** → clears `dw_is_at_home` and `dw_block_location` quest facts → **reload your save** → toggle OFF
* Shows a notification confirming the old and new values
* Available in the **Troubleshooting** category in Mod Settings

## Bug Fix: Context-Aware Combat Outfit Fallback

* **Problem**: When no location-specific or context-specific combat outfit matched, the generic fallback would randomly pick from *all* combat outfits — including context-specific ones like "combat corpo". This meant "combat corpo" could be randomly applied during combat in non-corpo areas
* **Fix**: The generic combat fallback now filters out outfits that contain context keywords (corpo, street, wild, club). Only truly generic combat outfits (e.g. "combat", "combat heavy", "combat gear") are eligible for the fallback

## Technical Details

* **Club outfit matching** — uses a list-swap pattern (same as combat outfits): temporarily restricts the outfit pool to `club`-prefixed outfits, then runs the location → context matching chain within that pool. Three tiers: (1) club + location keyword, (2) club + context keyword, (3) generic club fallback
* **Entrance detection** — each club has one or more entrance coordinates with per-entrance radius (1.0–15.0 units). Two detection types: Tier 1 (location enum + proximity) and Tier 2 (scene tier change + proximity, for elevators/loading zones)
* **Unique place IDs** — each club has a unique integer ID (10+) persisted in `dw_is_at_home`. On save/load, the ID is resolved back to the full club definition for precise outfit matching
* **`dw_is_at_home` fact values** — `0` = none, `1` = vanilla apartment, `2` = custom apartment, `10+` = unique place ID
* **Safety fallback exit** — if the location enum changes to a non-club district while V is flagged as being at a club, the club state is cleared (handles fast travel, free fly)
* **Enum flicker guard** — prevents false exits from the game flickering between parent district and club-specific enums
* **Context injection** — injects `"club"` into the active context list during matching, ensuring "club street" works at Little China clubs even after save/load
* **Combat not skipped** — club state uses a separate `m_wardrobeIsAtPlace` flag (not `m_wardrobeIsAtHome`), so combat applies normally
