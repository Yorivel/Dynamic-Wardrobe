# v1.2.1

## 🐛 Bug Fix: Manual Outfit Change Detection
- **Dedup guard now checks equipped state** — if you manually change your outfit via the Equipment-EX menu, the mod correctly detects the change and re-applies the appropriate context or location outfit when you move to a new area
- Previously, manually swapping outfits could cause the mod to skip re-applying because it thought the same context was still active

## 🔧 Technical Details
- **`ApplyOutfitByLocation` and `ApplyContextOutfit`** dedup guards now verify `outfitSystem.IsEquipped(player.m_wardrobeLastOutfit)` in addition to comparing `m_wardrobeLastContext` — if the tracked outfit is no longer equipped (manual EX change), the guard fails and the method re-applies
- **`ApplyOutfitByPrefix` intentionally unchanged** — some prefix contexts (`home`, `nude`, `ripper`) are used during scene transitions where the engine can temporarily strip items; adding `IsEquipped` there would cause false failures and spurious re-applies mid-scene
