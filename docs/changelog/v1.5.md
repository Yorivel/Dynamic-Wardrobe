# v1.5

## Update Notice

This version reorganizes the file structure. **Delete the old `Dynamic Wardrobe` folder entirely before installing v1.5** — leftover files from v1.4 (e.g. `HomeDetection.reds`, `SceneRestore.reds`) will cause Redscript compilation errors. If you've customized `Core/ContextMap.reds` or `Core/CustomApartments.reds`, back them up first and restore your changes into the new `UserConfig/` folder after updating.

## Bug Fix: Corpo Plaza Apartment Detection

* **Problem**: The Corpo Plaza apartment has an unusually small trigger volume (doorway only) and sits at the intersection of 3 different location enums (`CorpoPlaza`, `Downtown`, `CityCenter`). If the location callback fires after V walks past the 1.0m entrance threshold, the mod falls through to location/context matching instead of detecting home entry — applying a "corpo" outfit instead of the home outfit
* **Fix**: Added an apartment enum fallback in the location callback. If the enum matches a known apartment (`IsKnownApartment`) but V isn't near the entrance coordinates, the mod still recognizes it as a home entry. Proximity remains the primary detection path; the enum check is a safety net that catches timing-dependent misses

## Bug Fix: Reserved Prefix Keyword Collision

* **Problem**: Outfits named with reserved prefixes plus location/context keywords (e.g. `home corpo`, `combat street`) could leak into the normal location/context matching pool, causing them to be applied outside their intended context
* **Fix**: `ApplyLocationOrFallback` now filters all reserved prefixes (`home`, `nude`, `ripper`, `combat`) from the matching pool before running the location → context → outdoor fallback chain. Previously only `combat` was filtered

## Internal Refactoring

### OutfitManager Split

* **`OutfitManager.reds`** (515 → ~300 lines) split into three focused files
* **`LocationMatcher.reds`** — location enum parsing, bidirectional keyword matching, `HasOutfitForLocation`, `ApplyOutfitByLocation`
* **`ContextMatcher.reds`** — context keyword scoring, `GetContextOutfits`, `HasContextOutfit`, `ApplyContextOutfit`
* `OutfitManager.reds` retains prefix matching, EX sync helpers, the fallback chain, and combat outfits. `SplitOutfitKeywords` and `PickRandom` promoted to public (shared by both matchers)

### File Renames

* `Events/HomeDetection.reds` → `Events/LocationHandler.reds` — better reflects its role as the main location event router, not just home detection. Static class `HomeDetection` renamed to `LocationHandler`
* `Events/SceneRestore.reds` → `Events/SceneHandler.reds` — consistent with the `*Handler.reds` naming convention

### Config File Reorganization

* `Core/ContextMap.reds` and `Core/CustomApartments.reds` moved to `UserConfig/` folder — clearly separates user-editable config from engine logic. **Back up these files before updating** — new versions will overwrite them

## Technical Details

* **Apartment enum fallback** — in `OnWardrobeLocationChanged`, the else branch (not near entrance) now checks `IsKnownApartment(value)` before falling through to `ApplyLocationOrFallback`. The apartment enum string alone is sufficient proof of home entry — the game only reports apartment-specific enums inside their trigger volumes
* **`IsReservedPrefix`** — new helper in `OutfitManager` that checks for home/nude/ripper (via `StrBeginsWith`) and combat (via `StrContains`). Used by `ApplyLocationOrFallback` to filter reserved outfits from the matching pool
